# Frontend Module 8: PWA, Admin Dashboard, Accessibility & Performance

## Overview

This module adds Progressive Web App capabilities, a full admin dashboard, platform-wide accessibility improvements, and performance optimisations to NeuronHire.

## 1. PWA Setup

### Web App Manifest (`/public/manifest.json`)
- `name`: "NeuronHire", `short_name`: "NeuronHire"
- `theme_color` / `background_color`: `#080B14` (matches `--bg-base`)
- `display`: standalone, `start_url`: /dashboard
- Icons: 72, 96, 128, 144, 152, 192, 384, 512px + maskable 192/512
- App shortcuts: Browse Bounties, My Wallet
- Screenshots for app-store display (desktop + mobile)

### next.config.js ‚Äî next-pwa integration
- Service worker generated by `next-pwa@5.6.0`
- **Network-first** for `/api/*` calls (10s timeout, 30s cache)
- **Cache-first** for static assets (JS, CSS, fonts, images ‚Äî 30-day TTL)
- **Offline fallback**: `/offline` page served when network unavailable
- SW scope header: `Service-Worker-Allowed: /`
- PWA disabled in development (`NODE_ENV === 'development'`)

### Push Notification Service Worker (`/public/sw-push.js`)
Handles background push events with typed notification payloads:

| Type | Title pattern | Actions |
|------|--------------|---------|
| `bounty` | "New bounty: [Title] ‚Äî ‚Çπ[amount]" | View Bounty, Dismiss |
| `payment` | "Payment released: ‚Çπ[amount] from [Company]" | View Wallet, Dismiss |
| `message` | "New message from [Name]" | Reply, Dismiss |
| `proposal` | "Your proposal was accepted!" | View Contract, Dismiss |

Notification click handler navigates to the correct page or focuses an existing window.

### Install Prompt (`/src/components/pwa/install-prompt.tsx`)
- Tracks visit count in `localStorage` (`nh_visit_count`)
- Shows after **2nd visit**, never if dismissed (`nh_install_dismissed`) or installed (`nh_install_done`)
- **Mobile**: bottom slide-up sheet with drag handle
- **Desktop**: top banner that slides down
- Intercepts `beforeinstallprompt` to show custom UI before browser default
- Dismissal and install outcome both persisted to `localStorage`

### Push Notification Prompt (`/src/components/pwa/push-notification-prompt.tsx`)
- Custom pre-prompt modal shown **before** the browser's native permission dialog
- Triggered at high-intent moments (e.g. after passing assessment)
- Shows 3 notification preview examples to set expectations
- `usePushNotifications()` hook manages permission state and VAPID subscription
- On accept: calls `Notification.requestPermission()` then POSTs subscription to `/api/push/subscribe`

### Root Layout Updates (`/src/app/layout.tsx`)
- `<link rel="manifest" href="/manifest.json">` via Next.js `metadata.manifest`
- Apple PWA meta tags: `apple-mobile-web-app-capable`, `apple-touch-icon`
- `viewport` export with `themeColor: '#080B14'`
- `<InstallPrompt />` rendered globally

### Offline Page (`/src/app/offline/page.tsx`)
- CSS-only disconnected illustration (concentric rings + WiFi-off SVG + orbital dots)
- "You're offline. Check your connection." message
- "Try Again" button (`window.location.reload()`)
- "Go to Home" link
- Fully styled in NeuronHire design system

## 3. Accessibility

Applied platform-wide via `globals.css` and component updates.

### Focus Management
- `:focus-visible` ring: `2px solid var(--accent-cyan)` ‚Äî never removed
- `:focus:not(:focus-visible)` removes outline for mouse users only
- All modals: `role="dialog"` + `aria-modal="true"` + `aria-label` or `aria-labelledby`
- Modal focus trap handled by the existing `Modal` component (Escape closes)

### ARIA Labels
- All icon-only buttons have `aria-label`
- Chart containers: `role="img"` + `aria-label` describing the chart
- Progress bars: `role="progressbar"` + `aria-valuenow` + `aria-valuemin` + `aria-valuemax`
- Activity feed: `role="log"` + `aria-live="polite"`
- Filter button groups: `role="group"` + `aria-label`
- Filter buttons: `aria-pressed` reflecting current state
- Tables: `aria-label` on `<table>` elements
- Bulk action toolbar: `role="toolbar"` + `aria-label`

### Reduced Motion (`globals.css`)
```css
@media (prefers-reduced-motion: reduce) {
  /* All animations disabled ‚Äî duration set to 0.01ms */
  /* Blobs, marquee, pulse-ring, skeleton shimmer all stopped */
  /* Modals get a simple opacity fade instead of slide */
}
```

### Skip Link
`.skip-to-main` class: visually hidden until focused, then slides into view at top of page.

### ARIA Busy
`[aria-busy="true"]` gets `cursor: wait` + reduced opacity for loading containers.

### Features
- Opens with `Cmd+K` (Mac) or `Ctrl+K` (Windows/Linux) ‚Äî `useCommandPalette()` hook
- Fuzzy search across label, description, keywords, and group
- Grouped results: Navigate | Company | Admin
- Keyboard navigation: `‚Üë‚Üì` to move, `‚Üµ` to select, `Esc` to close
- Mouse hover also updates selection

- Input: `role="combobox"` + `aria-expanded` + `aria-autocomplete="list"` + `aria-controls` + `aria-activedescendant`

## 5. Performance

### Dynamic Imports (`/src/lib/dynamic-imports.ts`)
All heavy dependencies are code-split:

| Export | Package | SSR |
|--------|---------|-----|
| `LazyMonacoEditor` | `@monaco-editor/react` | `false` |
| `LazyAreaChart` | `recharts` | `false` |
| `LazyLineChart` | `recharts` | `false` |
| `LazyBarChart` | `recharts` | `false` |
| `lazyConfetti()` | `canvas-confetti` | n/a (function) |
| `LazyCommandPalette` | `./command-palette` | `false` |
| `LazyHireModal` | hire-modal | `false` |
| `LazyTrialModal` | trial-modal | `false` |

### Bundle Analyser
```bash
npm run analyze   # ANALYZE=true next build ‚Äî opens bundle visualiser
```

### Image Optimisation
- `formats: ['image/webp']` enforced in `next.config.js`
- All `next/image` usage gets WebP automatically
- `deviceSizes` configured for responsive images

### CSS Performance Utilities
- `.cv-auto` ‚Äî `content-visibility: auto` for off-screen sections
- Explicit `aspect-ratio` classes (`.aspect-profile`, `.aspect-card`, `.aspect-thumbnail`) prevent CLS
- `img { max-width: 100%; height: auto; }` base reset

### Core Web Vitals Targets
| Metric | Target | Strategy |
|--------|--------|---------|
| LCP | < 2.5s | `priority` on hero images, preconnect for fonts |
| CLS | < 0.1 | Explicit image dimensions, skeleton sizes match final layout |
| INP | < 200ms | No heavy sync work on interaction handlers, dynamic imports |
